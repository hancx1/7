# 健身打卡

> 《移动终端程序设计》结课项目 · 基于 H5 与 Vue 的运动打卡应用

一款记录日常运动的手机应用：每天打卡运动类型与时长，统计连续打卡天数与本周运动量。
采用「前端 H5（Vue 3）+ Node 后端 + JSON 文件持久化」结构。

## 技术栈

| 层次 | 技术 |
|------|------|
| 前端 | **Vue 3**（本地引入，无需联网）+ 原生 CSS（含纯 CSS 柱状图） |
| 后端 | Node.js 内置 `node:http`（CommonJS，**面向对象 class** + 正则数组路由） |
| 数据存储 | **JSON 文件** `data.json`（课程允许「可以使用文件」） |
| 依赖 | **无任何第三方 npm 依赖**（Vue 以本地文件引入） |

## 运行方法

环境要求：**Node.js ≥ 18**。无需 `npm install`。

- **方式一（推荐）**：双击 `启动演示.bat`。
- **方式二**：命令行执行 `node server.js`，浏览器打开 <http://localhost:3002/>。

> 手机效果：Chrome 中按 `F12` → 切换设备工具栏（`Ctrl+Shift+M`）。

## 功能

1. **打卡**：选择运动类型（跑步/骑行/游泳/健身房/瑜伽/球类/步行）、时长、强度、备注并提交。
2. **今日首页**：显示连续打卡天数（streak）、今日打卡次数与时长、本周运动量、最近打卡。
3. **历史记录**：按日期倒序查看全部打卡，可删除。
4. **统计**：本周每日运动时长柱状图 + 各运动类型时长占比 + 累计数据。

## 数据持久化

所有打卡记录以 JSON 数组形式保存在 `data.json`（字段：运动类型、时长、强度、备注、日期等），
每次新增/删除都会写回文件，重启服务后数据仍在。

## 接口一览

| 方法与路径 | 说明 |
|------------|------|
| `GET /api/checkins` | 查询全部打卡（日期倒序） |
| `POST /api/checkins` | 新增一条打卡 |
| `DELETE /api/checkins/:id` | 删除一条打卡 |
| `GET /api/stats` | streak、今日/本周数据、各类型占比等统计 |
